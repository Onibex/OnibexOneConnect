# SAP Transactions and Roles Required for OneConnect

The following transactions and permissions must be available in the SAP system before deploying and operating OneConnect. Please coordinate with your **SAP Basis team** to ensure all access is granted prior to the installation.

---

## OneConnect Transactions

These are the custom transactions delivered by Onibex as part of the OneConnect installation package.

| Transaction | Description |
|---|---|
| **`ZONT_ONECM`** | Data Modeler. Main interface to design and manage SAP Data Products. |

---

## OneConnect Configuration Transactions

These standard SAP transactions are required to configure the OneConnect environment.

| Transaction | Description |
|---|---|
| **`SM30`** | Access to `ZONT*` custom tables for OneConnect configuration. |
| **`SNUM`** | Configure number ranges for `ZON*` objects. |

> ℹ️ For step-by-step instructions on configuring the number ranges, see the [Configure Z Range manual](./10_Configuring_ID_Ranges.md).

---

## Support and Validation Transactions

These transactions are used during installation, transport management, and functional testing of OneConnect.

| Transaction | Description |
|---|---|
| **`SE16N`** | Validate records in tables. |
| **`SLG1`** | Review application logs generated by OneConnect processes. |
| **`SM59`** | Create and manage HTTP connections (RFC destinations). |
| **`SE80`** | Review Onibex repository objects and access `ZON*` packages. |
| **`STMS`** | Install Onibex transport requests. |
| **`SE09` / `SE10`** | Manage transport requests. |
| **`VA01` / `VA02`** | Create and edit **Sales Orders** (functional testing). |
| **`VL01N` / `VL02N`** | Create and edit **Deliveries** (functional testing). |
| **`VF01` / `VF02`** | Create and edit **Billing Documents** (functional testing). |
| **`ME21N` / `ME22N`** | Create and edit **Purchase Orders** (functional testing). |
| **`MIGO`** | Post **Inventory Movements** (functional testing). |
| **`XD01` / `XD02`** | Create and edit **Customer Master Data** (functional testing). |
| **`MM01` / `MM02`** | Create and edit **Material Master Data** (functional testing). |

> ℹ️ For detailed instructions on configuring RFC destinations in `SM59`, see the [Endpoint Customization (SM59) manual](./15-Endpoint_Customization_SM59.md).

## Real-Time Event Configuration Transactions

These transactions are required to activate and monitor the **BOR events** that trigger real-time data streaming from SAP to OneConnect.

| Transaction | Description |
|---|---|
| **`SWELS`** | Activate the event monitor for real-time event tracking. |
| **`SWEL`** | Review the event log to validate that events are being raised correctly. |
| **`SWETYPV`** | Activate BOR event type linkages required by OneConnect. |

> ℹ️ For a complete list of BOR events used by OneConnect across SAP modules, see the [Real-Time Events Reference](./BOR_and_RAP_Events/).

---

## Required Permissions and Roles

The following authorizations must be assigned to the user or role that will operate OneConnect in SAP. Please coordinate with your **SAP Basis** or **Security team**.

- **Developer key and developer role**: required to create and manage ABAP structures, dictionary objects, and custom table types delivered by Onibex.
- **Posting permissions in the Sales Area** to be used for testing: required to create documents such as Sales Orders, Deliveries, and Billing Documents.
- **Posting permissions in the Purchasing Organization** to be used for testing: required to create documents such as Purchase Orders and Goods Movements.

> 📌 **Note:** The developer key and developer role are necessary because OneConnect installs custom ABAP objects (tables, function modules, programs) into the SAP system. Without these permissions, the transport import will fail.

> 💡 **Tip:** If you need assistance identifying the correct Sales Area or Purchasing Organization for your environment, the Onibex team can guide you during the onboarding process.
